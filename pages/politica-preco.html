Política de Preço
Política de preço trata-se de um conjunto de estratégias que têm como objetivo determinar um padrão a ser seguido para preços e meios de pagamento, podendo trazer vantagens e flexibilidade aos seus planos de negócios. Hoje a plataforma trabalha com dois tipos de política de preços, padrão (DEFAULT) e personalizado (CLUSTER).


Política Default
São os preços base para a negociação com os clientes B2B. O preço de padrão é o preço inicial antes de quaisquer descontos ou condições especiais serem aplicados de acordo com a regra de negócio do Distribuidor.

Vale ressaltar que todos os clientes na plataforma que não estão atrelados a uma Política Cluster estarão automaticamente incluídos para Política DEFAULT. Como o próprio nome já sugere, trata-se da política padrão reconhecida pela plataforma e a partir desse ponto o Distribuidor pode optar por mantê-lo nessa política ou enviá-lo para uma política CLUSTER


Política Cluster
São preços personalizados para cada cliente ou grupo de clientes. Esses preços levam em consideração fatores como os preços especiais, condições de pagamento, relacionamento comercial e outros critérios específicos acordados entre as partes envolvidas.

Preços Clusterizados permite que o distribuidor aplique preços diferenciados para os seus clientes, devido a possibilidade de enviar inúmeras políticas de preços, cada política possui os produtos, tabelas de preços e grupo de clientes que deseja vincular. Uma vez que a plataforma permite trabalhar com várias políticas, é possível aplicar os preços diferenciados.


Regras de Envio
Envio das requisições
Os envios não devem ultrapassar 100 produtos dentro da mesma requisição, em casos onde será feita uma carga alta, é necessário quebrar em blocos de no máximo 100 produtos por requisição.

É de suma importância controlar o fluxo de envios. A plataforma sempre trabalha em forma de atualização para apenas do que foi enviado, portanto os produtos que não sofreram alteração, não devem ser enviados nas próximas requisições (enviar apenas o que sofreu alteração com relação às últimas requisições).
Inclusão de clientes
Um cliente só pode fazer parte de uma política de preço por vez, então é preciso que as políticas contenham os preços dos produtos que serão comercializados para aquele cliente respectivo.
Atualização de preços
As políticas trabalham em forma de atualização, ou seja: Caso o distribuidor envie a primeira requisição para política X contendo 50 produtos, e outra requisição contendo somente 1 produto, a política irá conter 51 produtos dentro da plataforma.
Isso vale para inclusão de clientes na política: Caso houver a necessidade de incluir clientes em políticas existentes, basta seguir o exemplo de inclusão de clientes, não há necessidade de repetir as requisições anteriores, pois sobrecarregará o tempo de integração, devido ao volume dos pacotes.
Dúvidas Frequentes
Como faço para mover clientes para as políticas Clusters?
É necessário apenas enviar a requisição com a inclusão do CNPJ. Assim o cliente já estará na tabela de preço atualizada. (O mesmo deve ocorrer para mudança de políticas)
Como faço para mover um cliente para a política Default?
Todos os clientes cadastrados na plataforma automaticamente visualizam os preços da política Default. O único caso de inclusão de cliente na política default que deve ser feito é quando um cliente estava em outra política e deseja ser movido novamente para default.
O envio segue o mesmo realizado para a política Cluster, a única diferença é que o ID da política atribuida será a DEFAULT.
É necessário enviar todos os preços novamente quando um novo cliente é incluído nela?
Para mover o cliente em uma nova política é necessário apenas o envio da requisição de inclusão respeitando o modelo padrão especificado no manual de integração.
Não há necessidade de enviar produtos e preços novamente, a plataforma sempre armazena a última requisição enviada e recomendamos que atualizações de novos clientes, produtos e tabelas de preços sejam realizadas conforme a necessidade para que esse processo não onere a fila e o tempo de processamento.
Não utilizo mais uma política de preço. O que devo fazer?
Nesses casos recomendamos que seja enviado o preço zerado para todos os produtos e suas condições de pagamento. Além disso, os clientes incluídos nela devem ser movidos para as suas novas respectivas tabelas de preços.
Como faço para que meu cliente tenha disponibilidade apenas em condições de pagamentos específicas?
Recomendamos o envio de uma política apenas com as condições desejadas e o mesmo seja incluído nela. Dessa forma ele só irá visualizar os preços daquela política em especifíco.
Não irei trabalhar mais com uma condição de pagamento. Como faço para indisponibiliza-la?
Só é necessário enviar preço zerado à condição desejada de todas as políticas de preço disponíveis na plataforma. Assim, esse método de pagamento não ficará mais disponível.
