Integração de Limite de Crédito
Conheça o fluxo do processamento na plataforma e tire suas dúvidas.
Consiste no envio de valores, fornecidos pelo distribuidor ou por alguma fintech contratada para disponibilizar a modalidade de compras em boleto faturado.

Quando uma empresa vende produtos para outra empresa, pode oferecer a opção de pagamento a prazo, permitindo que o cliente pague em uma data futura acordada. No entanto, para mitigar o risco de inadimplência e garantir que o cliente possua capacidade financeira para cumprir com o pagamento, a empresa fornecedora estabelece um limite de crédito para o usuário.

Uma vez que o cliente é aprovado e disponibilizado limite de crédito, é habilitado para ele essa nova modalidade, portanto o envio de limite deve ser feito por estratégia do seller, conforme o negócio e confiabilidade nos clientes estratégicos.
Boas práticas para o envio de requisições
  • Envio de limite de crédito É de suma importância o envio de limite de crédito para os clientes na plataforma. Desta forma ficará disponível o método de pagamento faturado para compras.
  • Atualização de limite de crédito Após a realização e faturamento do pedido a plataforma automaticamente retira o valor utilizado e atualiza o limite.
    Porém, para cancelamento de pedidos a plataforma não repõe o valor utilizado novamente. Nesse caso deve ser enviado uma nova requisição atualizada de limite de crédito após cancelar o pedido.
  • Limite dentro das requisições O distribuidor deve seguir as boas práticas de requisições para a plataforma, independente do serviço. Sendo assim, solicitamos que não ultrapasse 100 CNPJS em uma única requisição de limite de crédito.
Dúvidas Frequentes
Porque o cliente não consegue efetuar pedidos com boleto faturado?
O cliente obrigatoriamente precisa ter status APV(Aprovado) e Limite de crédito disponível. Sem uma das informações integradas, não será possível efetuar o pedido.
Como impossibilitar um cliente de não efetuar pedidos apenas com boleto faturado?
Neste caso recomendamos apenas o envio de limite de crédito 0 para este cliente. Assim ele poderá ainda realizar pedidos com o método à vista.
Onde o cliente consegue verificar o valor de limite de crédito disponível?
O valor de limite disponível para o cliente ficará disponível no carrinho ao selecionar a opção de pagamento e também na página do cliente em “Limite de Crédito”.
Dependendo da confiabilidade do cliente, deixamos que o mesmo realize pedidos com boleto faturado ainda que não tenha limite de crédito disponível. A plataforma possibilita a compra nesses casos?
Não! Uma vez que o cliente tem limite de crédito 0, ele não poderá efetuar mais nenhuma compra de boleto faturado. Fica a critério do distribuidor enviar mais limite para determinado cliente.
Como é feito o acompanhamento do pagamento dos boletos faturados?
A IFC não se responsabiliza pelos pagamentos para pedidos com boleto faturado. O distribuidor fica responsável por efetuar o envio do boleto e acompanhar se o mesmo foi pago corretamente. Lembrando que ainda que a IFC não acompanhe, precisamos receber o status dos pedidos na plataforma.
O cliente pode obter a opção de pagamento Vale Troca para reembolso de pedidos com boleto faturado?
Infelizmente, não! Como a Infracommerce não é responsável pelo pagamento para essa condição, não é possível solicitar ao SAC para gerar o vale troca com o valor do pedido realizado, apenas para pedidos com condições a vista. Lembrando que nesses casos o cliente deve tratar o reembolso diretamente com o distribuidor.
O cliente realizou a compra de um pedido com boleto faturado na plataforma e teve seu pedido cancelado. O valor do limite de crédito é restaurado?
Não! A plataforma atualiza automaticamente o valor do limite apenas quando o pedido é gerado. Caso o mesmo seja cancelado, o distribuidor deve efetuar um novo envio atualizado do limite de crédito para o cliente. Recomendamos que o envio seja realizado logo após o cancelamento do pedido.
Como é feito o acompanhamento e controle do limite de crédito enviado pelo distribuidor para a plataforma?
A plataforma recebe apenas as informações e reflete na loja de acordo com o enviado pelo distribuidor. Todo o controle e análise de crédito é de responsabilidade do mesmo.
Quais são as opções disponíveis para a plataforma caso o distribuidor não envie o limite de crédito?
O cliente ainda terá as opções de pagamento à vista disponíveis na plataforma caso seu status seja PEN ou APV. Sendo elas: Boleto Antecipado, Cartão de Crédito, PIX e Vale Troca.
Quais são as políticas da plataforma em relação ao uso do limite de crédito enviado pelo distribuidor?
A única política estabelecida na plataforma é que o cliente deve utilizar o limite para apenas os produtos comercializados pelo distribuidor que liberou o crédito para ele. Uma vez que o cliente possua limite positivo e os pedidos atendem o valor mínimo, ele pode realizar compras à vontade.
O Limite de crédito é restituído assim que o cliente efetuar o pagamento do boleto?
Tudo depende da sua regra de negócio, pois se trata de um processo gerenciado pelo lado do distribuidor. Assim, se optar por seguir com o procedimento dessa forma, deve ser enviado a atualização de limite de crédito para o cliente assim que o pedido for pago.
